# Clearance for Android — native Kotlin + Jetpack Compose port (implementation plan)

## Context
The finished native iOS app, **Clearance** (SwiftUI + SwiftData, in `Clearance/` and
`Clearance.xcodeproj/`), is being given an Android counterpart. Swift code can't run on Android,
so this is a fresh, feature-equivalent re-implementation in the modern Android stack:
**Kotlin + Jetpack Compose + Material 3 + Room**. The iOS app is the functional spec; the Android
app mirrors its behavior and visual identity (Takeoff/Landing sequences, Gym/Swim modules, phases,
progress, add/edit/delete/skip, reset, true-black evening palette, haptics).

Code is authored in the cloud session; the user builds/runs it in **Android Studio** on an
emulator or an Android phone. No Mac needed, no fees to sideload (Google Play is a one-time $25
only if publishing).

## Key decisions
- **Location:** new `android/` directory inside the existing `flight-checklist` repo, developed
  and pushed on **`main`**. The iOS project at the repo root is untouched.
- **Stack:** Kotlin, Jetpack Compose (Material 3), Room (persistence), DataStore Preferences
  (enabled modules + seed version, the analog of iOS `UserDefaults`), AndroidX `ViewModel` +
  `StateFlow`. **No DI framework** — a manual `Application`-scoped database + a small
  `ViewModelProvider.Factory`.
- **applicationId / package:** `com.krya1012.clearance`.
- **SDK:** `minSdk 26`, `compileSdk`/`targetSdk 35`, Kotlin 2.x with the Compose compiler plugin.
- **Source of truth to mirror exactly:**
  - Seed content (v4, incl. the new morning "Final Gear Check") → `Clearance/Models/SeedData.swift`
  - ViewModel logic (sections grouped module→phase, progress excludes skipped, CRUD, toggle/skip/
    reset, `.core` always on) → `Clearance/ViewModels/ChecklistViewModel.swift`
  - Colors (evening true-black `#000000`, teal tint `#2DD4BF`; morning amber `#F59E0B`/`#FBBF24`)
    → `Clearance/Helpers/Theme.swift`
  - Screen composition → `Clearance/Views/*.swift`

## Behavior mapping (iOS → Android)
- `@Observable` VM + computed `sections` → `ViewModel` exposing `StateFlow`; DAO returns `Flow`,
  combined with `selectedChecklist`/`enabledModules` flows so the list recomputes reactively.
- In-memory grouping module→phase→items, sorted by `sortOrder`/`phaseIndex`/`orderIndex`.
- `progress` = completed / non-skipped total; `.core` always enabled; optional modules persisted.
- Swipe: Compose `SwipeToDismissBox` for **delete**; **edit / skip / restore** via a trailing row
  overflow (⋮) menu — Compose lacks SwiftUI's clean bidirectional swipe.
- Evening = force a **true-black dark** Material color scheme regardless of system setting.
- Haptics on toggle/skip/module-toggle/complete(success)/reset via `performHapticFeedback`.

## Execution (commit per phase, mirroring the iOS build-out)
1. **Scaffold** — Gradle files, manifest, `ClearanceApplication`, `MainActivity`, theme, empty
   screen that compiles & runs. Extend root `.gitignore` (Android: `build/`, `.gradle/`,
   `local.properties`, `*.apk`, `*.aab`, `.cxx/`, `captures/`). Commit.
2. **Data layer** — enums, `@Entity`, converters, DAO, database, `SettingsStore`, `SeedData` (v4
   content copied from `SeedData.swift`). Commit.
3. **ViewModel** — `StateFlow` sections/progress, CRUD, toggle/skip/reset, module persistence,
   factory wiring. Commit.
4. **UI** — DashboardScreen, list, row, progress header, module chips, editor sheet, theme colors.
   Commit.
5. **Polish** — true-black evening switch, haptics, Compose animations, content descriptions
   (accessibility), adaptive launcher icon, and an `android/README.md` (build/run steps). Commit.

Then push to `main` (retry with backoff on network errors).

## Verification (on the user's machine — can't compile in the cloud session)
1. Open the **`android/`** folder in **Android Studio** (Hedgehog+); let Gradle sync (this also
   regenerates the wrapper jar). Accept any suggested SDK/AGP version.
2. Run on an emulator (e.g. Pixel, API 34) or a physical device with **USB debugging** enabled
   (Settings → Developer options). For a shareable build: `./gradlew assembleDebug` →
   `app/build/outputs/apk/debug/app-debug.apk` (installable with no developer fee).
3. Confirm parity with iOS: seed content loads (incl. morning Gym/Swim "Final Gear Check");
   Takeoff↔Landing switch updates list + palette (true-black evening); Gym/Swim chips inject/
   remove sections; tap toggles a row (dim + strikethrough) with haptic; reaching 100% shows
   "SEQUENCE COMPLETE ✓"; (+) adds, ⋮ menu edits/skips, swipe deletes; progress updates live;
   "Reset for tomorrow" clears both sequences after confirmation.

## Notes / caveats
- The Gradle **wrapper jar** is a binary that can't be authored as text; Android Studio creates it
  on first sync (or run `gradle wrapper` once with a local Gradle).
- One intentional UX divergence from iOS: edit/skip live in a row **⋮ menu** instead of a swipe,
  because Compose's swipe API doesn't cleanly support two-direction custom actions.
